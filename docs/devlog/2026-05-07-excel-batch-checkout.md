# 2026-05-07 Parts 批量操作 + CSV 导出

## 背景
用户需要批量管理零件。之前每次操作零件（如移动到新仓库）都要逐个点进详情页修改，效率很低。

另一个痛点是「装一架新飞镖机」场景：需要一次性扣除 BOM 清单中所有零件的库存，目前只能逐个点。

## 完成内容

### 1. CSV 导出功能（`src/lib/helpers.ts`）
- 新增 `exportPartsCSV()` 函数
- 在 `PartsPage` 页头增加「📥 导出」按钮
- 导出列：名称、型号/规格、分类、库存、单位、最低库存、存放位置(完整路径)、条码、供应商、备注
- BOM 编码，Excel 直接打开不乱码
- 文件名格式：`hamster-parts-YYYY-MM-DD.csv`
- 导出内容含当前筛选结果（关键词搜索 + 仓库筛选）

### 2. 批量操作（`src/pages/PartsPage.tsx`）
- 每个零件卡片左侧新增复选框
- 列表上方新增「全选当前列表」按钮
- 勾选零件后弹出蓝色操作栏，包含：
  - 已选零件数量显示
  - 目标位置选择器（树形下拉，支持仓库/货架/层位任意层级）
  - 📦 批量移动按钮
  - 取消选择按钮

### 批量移动流程
1. 通过搜索或仓库筛选找到目标零件
2. 勾选（可全选），弹出蓝色操作栏
3. 从树形下拉选择目标位置（如「新仓库 > A货架 > 3层」）
4. 点击「📦 批量移动」
5. Supabase 批量 UPDATE `location_id`，所有选中零件瞬间移动
