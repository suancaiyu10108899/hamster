## 一、项目背景

### 1.1 问题描述

RM 战队零件管理现状痛点：
- 零件种类多，规格杂，找零件靠记忆翻箱
- 买多了浪费，买少了要用时没有
- 不知道某个零件还有多少，要不要提前买
- 队友不知道有什么零件可用
- 零件放在哪个货架/哪个盒子，时间长了就忘了

### 1.2 用户画像

| 用户 | 角色 | 使用频率 | 主要场景 |
|------|------|---------|---------|
| 我 | 结构组成员 | 每天多次 | 入库、出库、查库存、找位置 |
| 队友（电控/机械） | 偶尔使用者 | 每周几次 | 查有没有某种零件、领料 |

### 1.3 核心目标

让 2 个人的零件管理从「翻箱凭记忆」变成「扫码即知」。

## 二、核心场景（用例）

### 场景 1：零件入库

1. 打开 Hamster
2. 搜索零件名
3. 如果已有 → 点入库 → 输入数量 → 完成
4. 如果没有 → 新建零件 → 填写名称/分类/规格/存放位置 → 完成
5. 库存 +N，队友那边实时看到

### 场景 2：零件出库（取用）

1. 打开 Hamster → 找到零件
2. 点「出库」→ 输入数量 → 完成
3. 库存 -N，低于最低预警线 → 首页标红提醒

### 场景 3：紧急找零件

1. 搜索关键词
2. 看到匹配零件列表
3. 点进去看存放位置
4. 拿着手机去货架找

### 场景 4：货架盘点（P1）

1. 扫货架/盒子上的二维码
2. 弹出该位置所有零件及应有数量
3. 对照实物核对，差异处直接修改

### 场景 5：比赛前 BOM 导出（P2）

1. 导出当前零件库存清单
2. 按分类分组 → 发队友确认

### 场景 6：采购一键入库（P1）

1. 打开「采购入库」页面
2. 逐行添加：选零件 → 填数量 → 填单价 → 自动算小计
3. 总价自动求和，填采购日期和购买链接
4. 点「确认入库」→ 所有零件库存自动增加，采购记录保留

### 场景 7：按替代型号浏览（P2）

1. 搜索某零件
2. 详情页显示「替代型号」区域
3. 列出组内其他零件 + 库存

## 四、功能详述

### 4.1 零件管理

一个零件包含的信息：

| 字段 | 必填 | 实现状态 |
|------|:--:|:--:|
| 名称 | ✅ | ✅ 已实现 |
| 分类 | ✅ | ✅ 已实现（选择已有分类） |
| 当前数量 | ✅ | ✅ 已实现 |
| 最低库存 | ❌ | ✅ 已实现（字段） |
| 单位 | ❌ | ✅ 已实现 |
| 存放位置 | ❌ | ✅ 已实现（选择已有位置） |
| 条码 | ❌ | ✅ 已实现（文本字段） |
| 照片 | ❌ | ❌ |
| 备注 | ❌ | ✅ 已实现 |
| 供应商 | ❌ | ✅ 已实现 |

### 4.2 分类管理

- 支持无限级分类（parent_id 自引用）
- 预置 9 大分类：紧固件、电子元件、电机、传感器、线材、结构件、工具、耗材、其他

### 4.3 出入库操作

| 操作类型 | 对库存影响 | 状态 |
|---------|-----------|:--:|
| 入库 | +N | ✅ |
| 出库 | -N | ✅ |
| 报废 | -N | ❌ |
| 盘点修正 | 直接设置值 | ❌ |

### 4.4 自定义字段（EAV）

- 数据库设计已完成（custom_fields 表）
- UI 未实现
- 每种分类可预设字段模板

### 4.5 一键采购入库（P1）

- purchases + purchase_items 表已设计
- 前端 UI 未实现

### 4.6 零件替代关系（P2）

- part_groups + part_group_members 表已设计
- 前端 UI 未实现

## 术语表

| 术语 | 含义 |
|------|------|
| 零件 | 泛指任何可管理的物料 |
| 入库 | 增加库存 |
| 出库 | 减少库存（取用） |
| 报废 | 减少库存（损坏/丢失） |
| 盘点 | 核对实物与系统库存 |
| EAV | Entity-Attribute-Value，自定义字段的数据库模式 |
| 替代组 | 多个可互替零件的逻辑分组 |
