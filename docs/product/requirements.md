## 一、项目背景

### 1.1 问题描述

RM 战队零件管理现状痛点：
- [ ] **（待补充）** 零件种类多，规格杂，找零件靠记忆翻箱
- [ ] **（待补充）** 买多了浪费，买少了要用时没有
- [ ] **（待补充）** 不知道某个零件还有多少，要不要提前买
- [ ] **（待补充）** 队友不知道有什么零件可用，来问我
- [ ] **（待补充）** 零件放在哪个货架/哪个盒子，时间长了就忘了

### 1.2 用户画像

| 用户 | 角色 | 使用频率 | 主要场景 |
|------|------|---------|---------|
| 我 | 结构组成员 | 每天多次 | 入库、出库、查库存、找位置 |
| 队友（电控/机械） | 偶尔使用者 | 每周几次 | 查有没有某种零件、领料 |

### 1.3 核心目标

让 2 个人的零件管理从"翻箱凭记忆"变成"扫码即知"。

## 二、核心场景（用例）

### 场景 1：零件入库

**触发条件**：新买了一包 M3x10 内六角螺丝（10个）

**操作流程**：
1. 打开 Hamster
2. 搜索"内六角 M3"
3. 如果已有该零件 → 点入库 → 输入数量 10 → 完成
4. 如果没有 → 新建零件 → 填写名称/分类/规格/存放位置 → 完成

**期望结果**：库存 +10，队友那边实时看到

### 场景 2：零件出库（取用）

**触发条件**：做机构需要 4 个 M3x10 螺丝

**操作流程**：
1. 打开 Hamster → 找到"M3x10 内六角螺丝"
2. 点"出库" → 输入 4 → 完成

**期望结果**：
- 库存 -4
- 如果库存低于最低预警线 → 标红提醒"该进货了"

### 场景 3：紧急找零件

**触发条件**：急需一个 M3 螺丝，但不知道具体放在哪里

**操作流程**：
1. 搜索"M3"
2. 看到所有 M3 相关零件列表
3. 点进去看存放位置
4. 拿着手机去货架找

**期望结果**：5 秒内找到位置信息

### 场景 4：货架盘点

**触发条件**：站在货架前，想确认这个盒子里有哪些零件

**操作流程**：
1. 扫一下货架/盒子上贴的二维码
2. 弹出该位置所有零件列表及应有数量
3. 对照实物逐一核对
4. 差异处可以直接修改数量

**期望结果**：手机一扫，库存一目了然

### 场景 5：比赛前 BOM 导出

**触发条件**：比赛前需要确认物料清单

**操作流程**：
1. 导出当前所有零件库存清单
2. 按分类分组
3. 发给电控和机械组确认

**期望结果**：一键导出 CSV/Excel，不用手工整理

### 场景 6：采购一键入库（P1）

**触发条件**：淘宝买了一堆零件——MG90S ×5 单价¥12、MG996R ×3 单价¥25、M3螺丝×100 单价¥0.05。总价¥140。

**操作流程**：
1. 打开"采购入库"
2. 逐行添加零件：搜索/选择零件 → 填数量 → 填单价 → 自动计算小计
3. 最下面显示总价，填采购日期和购买链接
4. 点"确认入库" → 所有零件库存自动增加，采购记录保留

**期望结果**：一次填完所有采购明细，一键全部入库。后续可查历史采购记录。

### 场景 7：按替代型号浏览（P2）

**触发条件**：做拨弹机构需要一个小扭矩舵机，MG90S、SG90、ES08MA 都行。

**操作流程**：
1. 搜索"MG90S"
2. 在零件详情页看到"替代型号"区域
3. 列出 SG90（库存 3）、ES08MA（库存 1）
4. 可以比较库存和价格，决定用哪个

**期望结果**：不需要跨越分类去找兼容零件，一个页面看清所有可选项。

## 三、功能需求

### 3.1 功能优先级

#### P0（第一版 MVP，必须实现）

- [ ] 零件增删改查（名称、分类、数量、存放位置、备注）
- [ ] 分类管理（支持无限级自定义树形分类）
- [ ] 出入库操作（入库 +N / 出库 -N / 报废）
- [ ] 存放位置管理（支持录入和修改）
- [ ] 搜索（按名称、备注搜索）
- [ ] 手机端友好（响应式布局，大按钮，少打字）

#### P1（第二版，核心体验提升）

- [ ] 自定义字段/参数（不同零件有不同的规格字段）
- [ ] 库存预警（最低库存设置，低于阈值标红提醒）
- [ ] 操作历史记录（谁在什么时候做了什么）
- [ ] 零件拍照（一张图胜过千言）
- [ ] 扫码出入库（二维码关联零件/存储位）
- [ ] **一键采购入库**（填型号/数量/单价/链接，一次全部入库）

#### P2（第三版，锦上添花）

- [ ] BOM 导出（CSV/Excel）
- [ ] 数据统计（消耗趋势、常用零件排行）
- [ ] 供应商信息管理
- [ ] 条码/二维码生成（给没有码的零件批量生成）
- [ ] 离线支持（PWA，有网时自动同步）
- [ ] **零件替代关系**（兼容互替型号浏览）

## 四、功能详述

### 4.1 零件管理

**一个零件包含哪些信息：**

| 字段 | 必填 | 说明 | 示例 |
|------|:--:|------|------|
| 名称 | ✅ | 零件名称 | "M3x10 内六角螺丝" |
| 分类 | ✅ | 所属分类 | "紧固件 > 螺丝" |
| 当前数量 | ✅ | 库存数量 | 56 |
| 最低库存 | ❌ | 预警阈值 | 10 |
| 单位 | ❌ | 计量单位 | 个/包/米/卷 |
| 存放位置 | ❌ | 物理位置 | "A-03-02"（A架3层第2格） |
| 条码 | ❌ | 二维码/条码内容 | 自动生成或扫描录入 |
| 照片 | ❌ | 零件照片 | 拍照或上传 |
| 备注 | ❌ | 其他说明 | "M3螺丝有两种，注意区分" |
| 供应商 | ❌ | 购买来源 | "淘宝XX店" |

### 4.2 分类管理

- **支持无限级分类**（数据库 `parent_id` 自引用天然支持任意深度）
- 前端默认折叠显示两级，点击展开下级
- 可完全自定义分类树，无层级限制
- 预置常用分类：

  ```
  紧固件
    ├── 螺丝
    │     ├── 内六角
    │     └── 沉头
    ├── 螺母
    └── 垫片
  电子元件
    ├── 电阻
    ├── 电容
    └── IC
  电机
    ├── 舵机
    ├── 步进电机
    └── 无刷电机
  传感器
  线材
  结构件
    ├── 铝型材
    └── 3D打印件
  工具
  耗材
  其他
  ```

### 4.3 出入库操作

| 操作类型 | 对库存影响 | 场景 |
|---------|-----------|------|
| 入库 | +N | 新买了零件 |
| 出库 | -N | 拿零件去用 |
| 报废 | -N（不计入消耗） | 坏了、丢了 |
| 盘点修正 | 直接设置值 | 实际数量和系统不符时修正 |

### 4.4 操作记录

每次出入库记录：
- 时间
- 操作人
- 操作类型（入库/出库/报废/修正）
- 数量变化（+N / -N）
- 操作后库存
- 备注

### 4.5 自定义字段（EAV模式）

基础字段有限，不同零件需要不同的参数。例如：

| 零件类型 | 自定义字段示例 |
|---------|--------------|
| 螺丝 | 螺纹规格：M3, 长度：10mm, 头型：内六角, 材质等级：12.9 |
| 电机 | 额定电压：24V, 额定转速：500rpm, 编码器：1024线 |
| 传感器 | 协议：I2C, 量程：±16g, 精度：0.1% |

实现方式：
- 每种分类可以预设该分类的"模板字段"
- 新建该分类零件时自动带上这些字段
- 也可以为单个零件追加专属字段

### 4.6 一键采购入库（P1）

**场景**：每次采购要填型号、单价、数量、总价、购买链接。能不能一张表填完一键入库？

**设计**：
- `purchases` 表：一次采购记录（采购日期、总金额、购买链接、备注、操作人）
- `purchase_items` 表：采购明细（关联零件、数量、单价、小计金额）
- 确认入库时，系统为每个明细自动创建一条 `transactions(type='in')` 记录
- 零件库存自动更新
- 采购记录可回溯：什么时候买了什么、花了多少钱

**前端交互**：
- 一个"采购入库"页面
- 上部分是明细列表，逐行添加：选零件 → 填数量 → 填单价 → 自动算小计
- 下部分汇总显示"总金额 = ¥xxx"，填日期和链接
- 一个"确认入库"按钮，点击后原子性写入所有记录

### 4.7 零件替代关系（P2）

**场景**：做拨弹机构需要小扭矩舵机，MG90S、SG90、ES08MA 都可以。想一次性看到所有兼容型号的库存。

**设计**：
- `part_groups` 表：替代组（名称 + 说明，如"小扭矩舵机"）
- `part_group_members` 表：零件-替代组关联（每个成员可有替代说明，如"SG90 扭矩略小，成本更低"）
- 一个零件可以属于多个替代组
- 浏览某零件详情时，侧边显示"替代型号"列表（组内其他零件 + 各自库存 + 替代说明）

## 五、非功能需求

### 5.1 性能
- 零件列表加载 < 2 秒（零件总数 < 500 时）
- 搜索响应 < 1 秒

### 5.2 可靠性
- 数据不能丢（出入库操作必须原子化）
- 并发操作不冲突（两人同时操作同一零件时给出提示）

### 5.3 易用性
- 手机端优先设计（触摸友好）
- 核心操作（搜索、出入库）不超过 3 次点击
- 减少打字，多用选择/扫码

### 5.4 可维护性
- 代码分层清晰
- 有基本测试覆盖核心业务流程

## 六、非目标（明确不做）

- ❌ 多仓库/多战队管理
- ❌ 复杂的权限系统（审批流、角色管理）
- ❌ 财务功能（价格、报销、采购审批）—— 采购价格记录仅作参考，不做账
- ❌ 对接淘宝/京东等电商平台
- ❌ 自动生成采购订单
- ❌ AI 预测采购建议（P1 只做最低库存预警）
- ❌ 对接 RM 比赛系统

## 七、术语表

| 术语 | 含义 |
|------|------|
| 零件 | 泛指任何一种可管理的物料（螺丝、电机、传感器、线材、工具等） |
| 库存 | 当前持有的数量 |
| 入库 | 增加库存的操作 |
| 出库 | 减少库存的操作（取用） |
| 报废 | 减少库存的操作（损坏/丢失） |
| 盘点 | 核对实际库存与系统库存是否一致 |
| 存放位置 | 零件在物理空间中的位置描述 |
| 自定义字段 | 不同类型零件的专属规格参数 |
| 替代组 | 多个可互替零件的逻辑分组 |
| 采购入库 | 通过填写采购明细一次性批量入库 |
